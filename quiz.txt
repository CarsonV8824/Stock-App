Atomicity
All or Nothing
A transaction is treated as a single, indivisible unit
Either all its operations complete successfully, or none of them do, 
Prevents partial updates

Consistency
Valid States
Ensures a transaction moves the database from one valid state to another 
Follows all defined rules and constraints (like data types, uniqueness)

Isolation
Independence
Concurrent transactions don't interfere with each other
Result is the same as if they ran sequentially
Works even if transactions are ran simultaneously

Durability
Permanence
Once a transaction is committed, its changes are permanent
Changes survive system failures, crashes, or power loss

Why ACID Matters
Data Integrity: Prevents data corruption, duplicates, or inconsistencies, critical for financial or inventory systems
Reliability: Provides confidence that data is accurate and operations are predictable, even with multiple users
Business Trust: Enables smooth operations by minimizing data errors that could affect customers or partners

SQLite Column Types
NULL: The value is a NULL value
INTEGER: The value is a signed integer, stored in 0 to 8 bytes depending on the magnitude of the value
REAL: The value is a floating point value, stored as an 8-byte IEEE floating point number
TEXT: The value is a text string, stored using the database encoding (UTF-8, UTF-16BE or UTF-16LE)
BLOB: The value is a blob of data, stored exactly as it was input

SQLite Column Types (continued)
BOOLEAN: No separate Boolean storage class. They are stored as integers 0 (false) and 1 (true)
DATE and TIME: No separate storage class. They can be stored as TEXT, REAL, or INTEGER. https://sqlite.org/lang_datefunc.html

Create Table Statement
CREATE TABLE IF NOT EXISTS <tablename> (<column1> <type1>, <column2> <type2>, ...);
PRIMARY KEY AUTOINCREMENT ensures that the id will be unique and automatically incremented
NOT NULL ensures that the name column cannot be empty

SQL Injection
Code injection technique where attackers insert malicious SQL statements into entry fields to manipulate a database
Occurs when an application uses unvalidated or unsanitized user input to dynamically construct SQL queries
Allows the input to alter the query's structure

Insecure Insert Statement
# Error 1: Vulnerable to SQL Injection
# Bad practice: string interpolation for queries
sql_query = f"INSERT INTO students (name) VALUES ('{name}');"

# Error 2: Using executescript with user input
cursor.executescript(sql_query)

Secure Insert Statement
# Good: parameterized queries to prevent SQLi
sql_query = f"INSERT INTO students (name, age) VALUES (?, ?);"

# Good: execute not executescript for single query
cursor.execute(sql_query, [name, age])

Primary Key (PK)
A column that uniquely identifies each and every record (row) within a database table
Often an auto-incrementing integer

Composite Primary Key
A combination of columns that uniquely identifies each and every record (row) within a database table
Used when only one column won’t uniquely identify the records

Foreign Key (FK)
A column in one table (the "child" table) that creates a link to the primary key or a unique key in another table (the "parent" table)
Purpose is to enforce referential integrity and maintain consistent relationships between data across different tables

Select Using Joins (select.py)
# Fetch students enrolled in a specific course
sql_query = '''
        SELECT s.id, s.name, s.age
        FROM students s
        JOIN student_course sc ON s.id = sc.student_id
        WHERE sc.course_id = ?;
    '''

Joins
Used to combine rows from two or more tables based on a related column (or a logical relationship) between them
SELECT column_list
FROM table1
JOIN table2
ON table1.column_name = table2.column_name;

# Sort student records alphabetically
sql_query = '''
        SELECT name
        FROM students
        ORDER BY name ASC;
    '''
# Sort student records reverse alphabetically
sql_query = '''
        SELECT name
        FROM students
        ORDER BY name DESC;
    '''

ORDER BY
The ORDER BY clause is used to sort the result set of a SELECT statement in a specified order
It is the only way to guarantee a sorted result, as database systems do not inherently return rows in any particular order without it.
ASC -> ascending (smallest to largest, alphabetical)
DESC -> descending (largest to smallest, reverse alphabetical)
